# Intake Guide

Use this guide when the user's request is underspecified.

Ask only for the smallest set of details that will change the recommendation.
Do not mechanically ask every question.

## High-value intake fields

1. Topic or problem area
   - What kind of repo are we looking for?
   - Example: agent memory, browser automation, MCP server, observability

2. Desired output size
   - Top 5, Top 10, shortlist, broad landscape

3. Intended use
   - adopt directly
   - fork and extend
   - study reference implementations
   - benchmark the space

4. Preferred repo shape
   - product
   - framework
   - infrastructure layer
   - curated list
   - research or methodology

5. Minimum maturity
   - star floor
   - recent maintenance
   - non-archived

6. Technical preferences
   - language
   - stack
   - deployment style

7. Exclusions
   - tutorial repos
   - archived repos
   - paper-only repos
   - commercial wrappers

## Fast intake examples

### Minimal
- Topic?
- Are you looking for something to use directly, or a framework to build on?
- Want a quick top 5 or a broader top 10?

### More specific
- What topic should I search?
- Should I optimize for production-ready products, hackable frameworks, or reference architectures?
- Any stack preference, such as TypeScript, Python, or Go?
- Any minimum stars or recency requirement?

## When to skip follow-up questions

You can start immediately if the user already specified most of:
- topic
- intended use
- rough maturity threshold
- desired output size

In that case, proceed and state assumptions briefly.
